Flow of a web request:
    when a req arrives at a django app the incoming request url is compared to the list of paths in th eurls.py in the variable urlpatterns 

    when thre is a url match, it selects a "view" which is a bit of code that handle any database access and then produce and delivers the responce to the browser

    the view access database indirectly through an abstraction called model

    this is a general web pattern called "model-view-controller" or MVC



a user clicks on a button on the page, the browser intercept it and make  a socket connectoion to the backend server:
                    the server has some functions like:
                        routing: -> urls.py
                        views: -> views.py, forms.py
                        models: -> DBs -> model.py
